02 / 12 Kontekst

Bakgrunn

Lyse har tatt de første skrittene med Claude Enterprise.

Piloterfaring

Pilot gjennomført

Lyse har gjennomført en pilot med Claude Enterprise for 150 brukere, fordelt på tekniske og ikke-tekniske roller.

Gap identifisert

Behov for struktur

Piloten har gitt verdifull erfaring, men avdekket behov: governance, sikkerhet, opplæring og forvaltning er ikke på plass for full utrulling.

Oppdrag

Ekstern kompetanse

IT-organisasjonen har fått i oppdrag fra forretningen å hente ekstern kompetanse for å operasjonalisere og skalere.

Teknisk kontekst

Samspill og forvaltning

Samspillet med Microsoft 365 og GitHub Copilot må avklares. Målet er effektiv, trygg og bærekraftig bruk med et forvaltningsopplegg som kan videreføres.

04 / 12 Fase 1 / Uke 1–2

Oppstart

Vi etablerer et felles situasjonsbilde og legger grunnlaget for resten av prosjektet.

Kickoff med Lyse: forankring av mål, roller og arbeidsform
Gjennomgang av piloterfaring: hva fungerer, hva mangler, hva er risiko
Interessentavklaring: hvem påvirkes, hvem bestemmer, hvem må involveres
Oppstart av sikkerhet- og personverngjennomgang
Leveranser: felles situasjonsbilde, prioritert arbeidsplan og bekreftet prosjektmandat
05 / 12 Fase 2 / Uke 2–5

Design

Vi utformer rammeverket som gjør trygg og effektiv bruk av Claude mulig for Lyse.

Utforming av governance-modell: retningslinjer, roller, ansvar og eskalering
Gjennomføring av sikkerhet-, personvern- og regelverksgjennomgang
Teknisk tilrettelegging for Claude Code-miljøer: krav til arbeidspraksis og kontroll før bredding
Avklaring av samspill mellom Claude, Microsoft 365 og GitHub Copilot
Utvikling av opplærings- og informasjonsmateriell for trygg og effektiv bruk
Leveranser: godkjente rammeverk, retningslinjer og materiell klart for utrulling
06 / 12 Fase 3 / Uke 5–10

Etablering

Vi ruller ut Claude til alle brukergrupper på en kontrollert og strukturert måte.

Trinnvis utrulling til prioriterte brukergrupper, Claude Chat og Claude Code separat
Opplæring gjennomføres parallelt med utrulling
Løpende oppfølging og justering basert på tilbakemeldinger underveis
Kontrollpunkter før bredding sikrer at krav til teknisk tilrettelegging og arbeidspraksis er oppfylt
Leveranser: Claude Chat og Claude Code i aktiv bruk hos alle 150 brukere
07 / 12 Fase 4 / Uke 10–12

Overlevering

Vi sikrer en ryddig overgang til varig forvaltning i Lyses egen organisasjon.

Ferdigstilling og dokumentasjon av forvaltningsrutiner for Claude Enterprise
Forslag til målbilde for videre forvaltning, inkludert vurdering av tjenesteleveranse fra underleverandør
Overlevering til driftsorganisasjon
Evaluering av prosjekt og beslutning om videre samarbeid
Løpende forvaltning av Claude Enterprise vurderes innlemmet i eksisterende driftsavtale mellom Lyse og Sopra Steria
Leveranser: ferdig handover-pakke, forvaltningsmodell og innstilling til videre avtale

09 / 12 Teknisk kontekst

Samspill med eksisterende verktøy

Claude Enterprise skal fungere sammen med eksisterende arkitektur og verktøy.

Microsoft 365

Microsoft 365 og Copilot

Avklaring av overlapp og komplementaritet mellom Claude og Microsoft 365 Copilot. Retningslinjer for når man bruker hva, og hvordan verktøyene utfyller hverandre. Teknisk integrasjon og tilgangsstyring via Entra ID.

GitHub

GitHub Copilot

Avklaring av rollefordeling mellom Claude Code og GitHub Copilot for utviklere. Retningslinjer for kombinert bruk i utviklingsmiljøer. Sikring av at arbeidspraksis og kontrollpunkter dekker begge verktøy.

Prinsipp: Lyse skal ha én tydelig verktøystrategi, ikke tre parallelle AI-initiativ uten koordinering.
