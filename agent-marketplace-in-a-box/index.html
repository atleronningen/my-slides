1 — Sammendrag

AI-agenter i stor skala, under full kontroll

Kjerneprinsippet er at AI-agenter ikke lenger skal bygges og forvaltes i siloer. I stedet samles de i et felles, søkbart og versjonert register — på samme måte som applikasjoner distribueres fra en intern app store. Hver organisasjon får sin egen instans, tilpasset interne teknologivalg, sikkerhetspolicyer og AI-modellpreferanser.

Løsningen er modell- og plattformagnostisk. Den støtter ledende AI-plattformer fra Anthropic (Claude), OpenAI (GPT-serien), Google (Gemini), Microsoft (Azure OpenAI) og open-source-modeller via Hugging Face eller selvhostede tjenester. Integrasjonslaget er bygget på åpne standarder, inkludert Model Context Protocol (MCP).
2 — Løsningen

Hva er Agent Marketplace in a Box?

Plattformen samler AI-kapabiliteter som ellers er spredt på tvers av koderepo, dokumentasjonssider og individuelle arbeidsflater, og gjør dem tilgjengelige gjennom et enhetlig grensesnitt.

🤖

Agenter

Selvstendige AI-arbeidsenheter med et definert mål, et sett med tilgjengelige verktøy og instruksjoner for atferd. Kan automatisk innhente data fra CRM, strukturere det og produsere statusdokumenter.

🧩

Skills

Gjenbrukbare delkapabiliteter som monteres inn i agenter eller AI-assistenter. Kan inneholde domenespesifikk kontekst, prosjektkonvensjoner eller rolleprofiler som instruerer AI-modellen etter virksomhetens behov.

🔌

MCP-servere

Integrasjonspunkter som gir agenter tilgang til eksterne systemer og data i sanntid. Støttes av et raskt voksende økosystem av koblinger mot CRM, ERP, dokumentarkiver og utviklerverktøy.

📋

Instruksjoner og prompts

Systemprompter og oppgavespesifikke instruksjoner som sikrer konsistent atferd på tvers av bruksområder og modeller.

⚡

Kommandoer og hurtigmaler

Forhåndsdefinerte oppgaveutløsere som lar brukere aktivere komplekse agentarbeidsflyter med minimal konfigurasjonsterskel.

Referanseimplementasjon: Sopra Sterias interne Agent Marketplace er bygget rundt Claude-baserte agenter og gir et søkbart bibliotek organisert etter tjenestedomene, med mulighet for rask installasjon og tilpasning. Konseptet skalerer fra spesialiserte konsulenttjenester til store, teknologiintensive organisasjoner med over tusen utviklere.

3 — Teknisk arkitektur

Lagdelt og modulær arkitektur

Arkitekturen er designet for å integreres i kundens eksisterende teknologilandskap uten store strukturelle endringer. Løsningen kan deployes i kundens skyinfrastruktur (Azure, AWS, GCP), i en hybridmodell, eller on-premises.

Lag 1 Presentation Layer
Web-basert markedsplass-UI og API-endepunkter for programmatisk tilgang. Inkluderer søk, filtrering, kategorisering og installasjonsflyt for ressurser.
Lag 2 Registry & Catalog Layer
Versjonert ressursregister med metadata, taksonomi og tilgangsmodell. Støtter private, team-spesifikke og delte ressurser. Lagret i Git eller dedikert database.
Lag 3 Runtime & Orchestration Layer
Agentorkestrering som koordinerer modellkall, verktøybruk, kontekstvindu og kjøringsstatus. Støtter sekvensielle, parallelle og hierarkiske agentmønstre (multi-agent).
Lag 4 Integration Layer (MCP)
MCP-servere og tilpassede koblinger mot forretningssystemer. Standardisert autentisering via OAuth 2.0, API-nøkler og tjenesteprinsipal. Inkluderer rate limiting og retry-logikk.
Lag 5 Model Abstraction Layer
Adaptor-lag som abstraherer AI-modell-APIer bak et felles grensesnitt. Muliggjør modellbytte uten endring i agentlogikk. Støtter routing basert på oppgavetype, kostnad og kapasitet.

Typiske MCP-integrasjoner

Agenter kobles til eksisterende forretningssystemer via MCP-servere:

CRM — Salesforce, Dynamics, HubSpot
ERP — SAP, Oracle, Business Central
Dokumentarkiv — SharePoint, Confluence, Drive
DevOps — GitHub, Jira, Azure DevOps
Kommunikasjon — Teams, Slack, e-post
Databaser — JDBC/ODBC, REST-fasader

Deploymentmodeller

☁️ Fullt skybasert

Hostet i kundens skyabonnement (Azure, AWS eller GCP) med infrastruktur definert som kode (Terraform/Bicep). Enkel skalering og lavt driftskrav.

🔀 Hybrid

Markedsplassen og registeret hostes i skyen, mens utvalgte MCP-servere og agenter kjøres on-premises for å ivareta dataplassering og sikkerhetskrav.

🏢 Fullt on-premises

Komplett stack i kundens eget datasenter eller private sky, med AI-modeller hostet lokalt. Egnet for høysikkerhetsmiljøer og strengt regulerte sektorer.

4 — Nøkkelfunksjoner

Funksjonalitet som skalerer

Plattformen er bygget for å vokse med organisasjonen — fra pilotering av enkeltoppdrag til forvaltning av hundrevis av agenter i produksjon.

Ressursregister og distribusjon

Plattformens katalog er det sentrale knutepunktet for alle AI-kapabiliteter. Ressurser kan søkes opp via fritekst, filtreres og installeres med ett klikk direkte til støttede AI-arbeidsflater.

  • Installasjon til VS Code, JetBrains IDEer, Claude Code, GitHub Copilot og bedriftsinterne agentorkestreringer
  • Dokumentert formål, bruksanvisning og tekniske avhengigheter per ressurs
  • Kontaktperson, eierteam og godkjenningsstatus synlig i katalogen
  • Endringslogg og versjonshistorikk for full sporbarhet
  • Gjenbruk og transparens som designprinsipper for organisasjonstillit

Tilgangskontroll og forvaltningsmodell

Plattformen implementerer en rollebasert tilgangskontrollmodell (RBAC). Godkjenningsprosessen er konfigurerbar fra peer review til formell sikkerhetsvurdering.

🧑‍💼 Agent Owner
Ansvarlig for agents innhold, atferd og livssyklus. Godkjenner endringer.
⚙️ Platform Admin
Administrerer plattformkonfigurasjon, modellkobling og systemhelse.
👩‍💻 Developer
Kan opprette og publisere ressurser, underlagt godkjenningsprosess.
🔍 Consumer
Kan søke, installere og kjøre ressurser fra katalogen.
📋 Auditor
Lesetilgang til logger og kjøringsdata for compliance-formål.

Observabilitet og monitorering

Full kjøringssynlighet er innebygd. For hver agentjobb logges alle relevante data, eksponert til standard observabilitetsverktøy via OpenTelemetry.

  • Inndata og utdata, modell og versjon logges per jobb
  • Verktøykall med parametere, responser og kjøringstid
  • Kostnadsestimat (token-forbruk) og feilkoder
  • Integrasjon med Prometheus, Grafana, Azure Monitor og Datadog out of the box
  • Konfigurerbare varsler på feilrate, latens og kostnadsavvik

Versjonering og livssyklushåndtering

Alle ressurser versjoneres semantisk (major.minor.patch). Plattformen støtter parallell drift av versjoner og A/B-testing for gradvis utrulling.

  • Pinning til spesifikk versjon for reproduserbarhet i produksjon
  • Konfigurerbar latest-stable-tracking for automatiske oppdateringer
  • Parallell drift av flere versjoner med trafikkstyring
  • A/B-testing mellom agentversjoner for gradvis utrulling
  • Automatiske deprecation-varslinger basert på livssykluspolicyer

Sikkerhet

Sikkerhetsarkitekturen er bygget etter prinsippene om minste privilegium, forsvar i dybden og zero trust.

  • Autentisering: Integrasjon med Entra ID / Azure AD, Okta, SAML 2.0 / OIDC for SSO og MFA
  • Autorisasjon: RBAC kombinert med attributtbasert tilgangskontroll (ABAC)
  • Datahåndtering: Datamaskering og anonymisering i integrasjonslaget
  • Nettverkssikkerhet: TLS 1.3 på alle API-kall, MCP-servere i isolerte segmenter
  • Revisjonsspor: Uforanderlig logg egnet for SOC 2, ISO 27001 og NIS2-compliance

6 — Verdi og differensiering

Hva gjør løsningen unik?

Agent Marketplace in a Box differensierer seg fra rene SaaS AI-verktøy på fem sentrale dimensjoner. For tekniske beslutningstakere er det avgjørende at plattformen ikke introduserer nye siloer, men bryter dem ned.

Dimensjon Tradisjonell tilnærming Agent Marketplace in a Box
Kontroll og datasuverenitet ✕Data forlater organisasjonen via tredjepartsplattformer ✓Full kontroll over hvor data prosesseres og lagres
Gjenbruk ✕Agenter bygges isolert per team, ingen systematisk deling ✓Sentralt, søkbart bibliotek med versjonerte kapabiliteter
Modellfleksibilitet ✕Låst til én leverandørs modeller og API-grensesnitt ✓Bytt modell uten å endre agentlogikk — multi-provider
Forvaltningsmodell ✕Uklart eierskap, ingen systematisk livssyklushåndtering ✓Definerte roller, godkjenningsprosess og revisjonssyklus
Integrasjon ✕Punkt-til-punkt-integrasjoner som raskt blir uhåndterlige ✓Standardisert via MCP, sentralt administrert og dokumentert
7 — Utvidelsesmuligheter og veikart

Designet for inkrementell ekspansjon

Etter initial etablering kan følgende kapabiliteter legges til uten arkitektonisk ombygging.

🕸️

Multi-agent orkestrering

Samordning av spesialiserte agenter i komplekse, automatiserte arbeidsprosesser der agenter delegerer deloppgaver til hverandre.

⚡

Hendelsesdrevne agenter

Agenter som trigges av hendelser i forretningssystemer via meldingskøer og event streams — uten manuell aktivering.

🧑‍⚖️

Human-in-the-loop

Konfigurerbare godkjenningspunkter der agenter pauser og ber om menneskelig vurdering før kritiske handlinger utføres.

🧠

RAG og domenespesialisering

Integrasjon av RAG-pipelines og vektordatabaser for domene-spesifikk kunnskapsforankring uten modell-retraining.

🌐

Ekstern markedsplass-integrasjon

Import og tilpasning av ressurser fra offentlige markedsplasser og bransjespesifikke agentbiblioteker.

💰

Kostnads- og kapasitetsstyring

Avansert routing basert på realtidskapasitet, kostnad per token og SLA-krav, med automatisk fallback mellom leverandører.

8 — Oppsummering

Fra AI-eksperimenter til systematisk produksjonsdrift

Agent Marketplace in a Box er ikke ett verktøy, men en komplett teknisk og organisatorisk kapabilitet. Den gir organisasjoner en strukturert, sikker og skalerbar måte å gå fra eksperimenter med AI til systematisk drift av AI-agenter i produksjon.

Løsningen er bygget på åpne standarder, er modell- og skyagnostisk, og tilpasses kundens eksisterende teknologilandskap, sikkerhetspolicyer og regulatoriske rammeverk. Klar til å etableres i dag, designet for å vokse med organisasjonens AI-ambisjoner i årene fremover.
